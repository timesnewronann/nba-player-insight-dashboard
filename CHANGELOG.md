## [1.0.9] - 06-03-2026

### Added

- Shot chart visualization with D3.js supporting dots mode and hexbin zone mode
- getShotsByPlayerId in PlayerService.ts wired to GET /api/players/{id}/shots
- ShotChart TypeScript type mirroring the Java entity
- 132,015 shot chart rows loaded into shot_chart table via load_shot_charts.py
- game_event_id field added to ShotChart.java entity and shot_chart table (ALTER TABLE)
- vercel.json rewrite rule so all routes fall through to index.html, fixing 404 on page reload

### Changed

- CorsConfig.java switched from allowedOrigins to allowedOriginPatterns so wildcard https://*.vercel.app covers all Vercel preview and production URLs without updating env vars per deployment
- CORS_ALLOWED_ORIGINS Railway env var now drives allowed origins instead of hardcoded localhost
- ShotChartRepository replaced derived query method names with explicit @Query annotations navigating s.player.id and s.game.id, matching the pattern already used in PlayerGameStatRepository
- PlayerSeasonStat.java @Table annotation now declares composite unique constraint on (player_id, season) so ON CONFLICT in load_player_season_stats.py resolves correctly
- PlayerGameStat.java @Table annotation now declares composite unique constraint on (player_id, game_id) so ON CONFLICT in load_player_game_logs.py resolves correctly
- ShotChart.java @Table annotation now declares composite unique constraint on (player_id, game_id, game_event_id) so ON CONFLICT in load_shot_charts.py resolves correctly

### Fixed

- Railway healthcheck was hitting /api/players which requires a live database query — changed healthcheckPath to /actuator/health which is purpose-built for health monitoring and responds independently of data
- Player.java entity was missing @GeneratedValue(strategy = GenerationType.IDENTITY) causing players.id to be created as a plain bigint with no sequence, resulting in null value in column "id" errors during ETL — fixed annotation and ran ALTER TABLE players ALTER COLUMN id ADD GENERATED BY DEFAULT AS IDENTITY on the existing Railway table
- Team.java, Player.java, Game.java were missing unique = true on nba_team_id, nba_player_id, and nba_game_id columns respectively — Hibernate created the columns without UNIQUE constraints so ON CONFLICT clauses in ETL scripts had no constraint to target
- ShotChartRepository derived query findByPlayerIdOrderByGameDateDesc could not navigate the @ManyToOne player relationship and silently returned [] for all players — replaced with explicit JPQL @Query
- Vercel build failed with TS2339 because nbaPlayerId was used in PlayerPage.tsx but missing from the Player TypeScript type — added nbaPlayerId: number to Player.ts
